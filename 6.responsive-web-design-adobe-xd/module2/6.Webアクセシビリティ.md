# ウェブアクセシビリティについて
Webコンテンツを障がい者にとってよりアクセシブルなものにする  
[Web Content Accessibility Guidelines(WCAG)](https://www.w3.org/WAI/standards-guidelines/wcag/)を確認しましょう。  

## アクセシブルなマークアップ
### ナビゲーション順序
ナビゲーション順序は、  
スクリーンリーダー利用者やキーボード操作者が、ウェブサイトやアプリの操作をどのように行うかを示すための目印（マーカー）のことです。  
コール・トゥ・アクションボタンなどの操作可能なUI要素に付与されます。  
スクリーンリーダーやキーボードによる操作時に、要素間をタブキーで移動する順番を決定します。  
つまり、マウスやタッチスクリーンを使わないユーザーが、どのような順番でウェブサイトの要素を操作していくかを定義します。

アプリやウェブサイトのナビゲーション（画面の操作順序）は、画面の左上から右下へと進むように設計されています。
特に、この標準的なナビゲーション順序から外れるインタラクティブな要素（操作可能なボタンやリンクなど）や、  
ユーザーの最初の注目点が画面左上に配置された最初のインタラクティブ要素からずれるべき場合は、それらを明記することが重要です。  


キー操作の回数や、項目から項目へのタブ移動の回数を、数字で示すことができます。  
これらの数字は、下の画像で緑色の丸で表示されています。  
製品開発中、エンジニアはこの注釈も使用して、標準的なナビゲーション順序の変更を伝えます。

### Hierarchical headings
見出しはインターネットの言語であるHTMLで記述されます。  
H1、H2、H3といった特定の見出しタグを使用することで、スクリーンリーダーがページの階層構造をユーザーに解釈できるようになります。  
H1タグは特定の見出しがページ上で最も重要であることを示し、タイトルによく使用されます。  
H2タグはサブタイトルや注釈ボックスによく使われます。  
H3などの追加タグも、ページの階層構造をさらに示すために使用できます。

### Accessibility labels
ラベル（またはスクリーンリーダーによる音声化）は、ウェブページのインタラクティブなUI要素に説明文を追加します。  
ラベルがあれば、ボタン、メニュー、チェックボックスの目的が明確になります。  
例えば、「ユーザーがチェックボックスをクリックすると、項目が選択されます」といったラベルが考えられます。  
以下の画像の各ボタンについて、スクリーンリーダーが読み上げるべきラベルの追加例を確認してください。

ウェブサイトのデザインでインタラクティブ要素にラベルを付与する際は、  
効率性と文脈提供のため類似コントロールをグループ化する位置を考慮してください。  
例えば、支援技術利用者がチェックボックス間の関連性を理解し、素早く移動できるよう、チェックボックスのリストをグループ化することを検討しましょう。
エンジニアはこれらのラベルを基に、実現したい体験に最適な実装方法を決定します。

### Accessible color and contrast
輝度コントラスト比とは  
デザイン背景と使用される文字色のコントラストを測定する指標です。  
要するに、  
明るい背景に暗い文字、あるいは暗い背景に明るい文字を選択すれば、  
デザインのアクセシビリティは向上します。  
さらに、デザインで選択する色の組み合わせを慎重に検討する必要があります。  
色覚異常のある人は、赤と緑のような特定の色組み合わせの識別が困難な場合があります。  
また、特にコントラスト感度が低い人々は、薄い灰色と白のように色の組み合わせが似通っていると、  
個々の形状を区別できない可能性があります。
色を選択する際には、[WebAIMのコントラストと色の要件](https://webaim.org/articles/contrast/)を参照し、  
デザインがアクセシビリティ基準を満たしていることを確認できます。


### Make responsive websites accessible

異なる画面サイズに対応する際に、すべての表示形態でアクセシビリティを確保する必要があります。  
例えば、ユーザーが視覚的ニーズに応じて要素を拡大したりフォントサイズを大きくした場合、  
ページの残りの部分が自動的に再配置され、新しい画面サイズに適合すべきです。  
そうしないと、ユーザーがページの一部を拡大した際に要素が重なり合う可能性があります。

これらの異なるインタラクションをエンジニアに伝えることが極めて重要です。  
例えば、モックアップを複数バージョン作成すると効果的です。  
一つはデフォルトの表示設定とテキスト設定でのデザインを示すもの、  
もう一つは拡大表示（またはズームイン）状態を示すものです。  
これにより、ウェブサイトが完全に開発される前に、拡大表示やその他のレスポンシブデザインの要件を考慮に入れることができます。
