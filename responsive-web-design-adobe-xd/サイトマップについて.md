# サイトマップ(Sitemap)について
## サイトマップとは?
Webサイトやアプリの設計図で、以下を示すものです:

- ページがどう優先順位付けされているか
- ページ同士がどうリンクされているか
- ページがどうラベル付けされているか

## サイトマップの構造例
この構造は階層型モデルのサイトマップです。
```
      ┌────────────────┐
      │   ホームページ   │
      └─────────┬──────┘
                │
    ┌───────────┼──────────┬──────────┐
    │           │          │          │
┌───┴─────┐ ┌───┴────┐ ┌───┴────┐ ┌───┴────┐
│カテゴリ1 │ │カテゴリ2 │ │カテゴリ3│ │カテゴリ4 │
└───┬─────┘ └───┬────┘ └────┬───┘ └────┬───┘
    │           │           │          │
   ~~~ サブカテゴリ(縦に並ぶ長方形)   ~~~
```
## サイトマップを作る5つのメリット
### 1. 明確な組織構造の作成
Webサイトの初期段階で、整理された道筋を計画できます
### 2. 接続性の確認
- どのエリアがより複雑な接続を必要とするか分かる
- ユーザーが必要なページに到達できるか確認できる

### 3. 全体像の把握
すべてのページを一度に見ることで
- 忘れていた必要なページに気づける
- 削除や統合すべきページを見つけられる

### 4. 検索エンジンの最適化(SEO)
- Googleなどの検索エンジンがサイトをインデックス化する手助けをする
- サイトの内容を検索エンジンに伝える
- 検索結果に表示されやすくなる

💡 豆知識: Googleが検索結果ページにWebサイトをリストする方法を疑問に思ったことはありませんか? サイトマップがその答えです!
### 5. アクセシビリティの向上
#### なぜ重要か?
- あらゆる能力、アイデンティティ、経験を持つユーザーが成功裏にサイトを移動できる必要がある
- 支援技術を使用するユーザーにとって特に重要

#### メリット
- 組織化された流れで、ユーザーをタスクからタスクへ導く
- 支援技術を含むすべての訪問者が、より効率的にナビゲートできる

## まとめ
サイトマップは単なる「線と長方形の図」ですが、その価値は計り知れません:
- ✅ ユーザー体験の向上
- ✅ SEO対策
- ✅ アクセシビリティの確保
- ✅ デザインの早期検証
- ✅ チーム間のコミュニケーション改善
