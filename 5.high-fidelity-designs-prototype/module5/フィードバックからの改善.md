# ハイファイデリティデザインの改善プロセス
この文書では、ユーザビリティテストのフィードバックに基づいてデザインを改善する方法を説明しています。
## 📊 デザイン改善の2つの主要ステップ
### ステップ1: デザインに取り入れるリサーチインサイトを決定する
インサイトの優先順位付けをし、  
ユーザビリティテストから得られた発見を、以下の3つの優先度に分類します。

#### 🔴 P0 (Priority Zero) - 最優先事項
以下の必ず対応すべき問題を含むインサイト:

- ✗ 製品が正しく機能しない
- ✗ ユーザーが騙されたと感じる
- ✗ アクセシビリティ(利用しやすさ)に問題がある

例: 基本的な機能が動かない、重要な情報が見つからないなど、ユーザーが目的を達成できない根本的な問題

#### 🟡 P1 (Priority One) - 優先度高
ユーザー体験を大きく改善するが、必須ではないもの
```
例(犬の散歩アプリの場合):
ユーザーから「定期的な散歩の予約機能があればいいな」という要望
この機能がなくても予約はできるが、あれば便利になる
```

#### 🟢 P2 (Priority Two) - 優先度中
P0とP1の対応後に取り組むもの
```
例(ピザ注文アプリの場合):

「配達」ボタンを「持ち帰り」ボタンの左側に配置してほしい(より人気のあるオプションを目立たせるため)
ユーザーの好みに合わせた改善だが、後回しでも問題ない
```

## ステップ2: リサーチに基づいてデザインを更新する
### 1. アクションプランの作成
- まずP0インサイトに対応
- 次にP1インサイト、余裕があればP2インサイトに対応
- すぐにテストできるよう計画を立てる

### 2. モックアップとプロトタイプの更新
- ハイファイモックアップを修正
- 画面の順序を調整
- 正しいナビゲーションフローで画面を接続

### 3. ケーススタディやポートフォリオに追加
更新後、以下を作成:
- クライアント向けプレゼンテーションスライド
- ポートフォリオ用の事例紹介

## ✅ 重要なポイント

- 優先順位付けが鍵: P0 → P1 → P2の順で対応
- チームで協議: デザイナー、プロジェクトマネージャー、エンジニアと協力
- すぐにテスト: P0とP1の更新後、速やかにテストを実施
- 記録を残す: 改善のプロセスをポートフォリオに記録
